1005 继续(3n+1)猜想 (25分)

发布于 2020-04-02 | 分类于 代码练习 | 5分钟 | 1118字数

题目

卡拉兹(Callatz)猜想已经在1001中给出了描述。在这个题目里，情况稍微有些复杂。

当我们验证卡拉兹猜想的时候，为了避免重复计算，可以记录下递推过程中遇到的每一个数。例如对 n=3 进行验证的时候，我们需要计算 3、5、8、4、2、1，则当我们对 n=5、8、4、2 进行验证的时候，就可以直接判定卡拉兹猜想的真伪，而不需要重复计算，因为这 4 个数已经在验证3的时候遇到过了，我们称 5、8、4、2 是被 3“覆盖”的数。我们称一个数列中的某个数 n 为“关键数”，如果 n 不能被数列中的其他数字所覆盖。

现在给定一系列待验证的数字，我们只需要验证其中的几个关键数，就可以不必再重复验证余下的数字。你的任务就是找出这些关键数字，并按从大到小的顺序输出它们。

补充1001 卡拉兹(Callatz)猜想：

对任何一个正整数 n，如果它是偶数，那么把它砍掉一半；如果它是奇数，那么把 (3n+1) 砍掉一半。这样一直反复砍下去，最后一定在某一步得到 n=1。

输入格式：

每个测试输入包含 1 个测试用例，第 1 行给出一个正整数 K (<100)，第 2 行给出 K 个互不相同的待验证的正整数 n (1<n≤100)的值，数字间用空格隔开。

输出格式：

每个测试用例的输出占一行，按从大到小的顺序输出关键数字。数字间用 1 个空格隔开，但一行中最后一个数字后没有空格。

输入样例：

6
3 5 6 7 8 11

输出样例：

7 6

思路：

题目难度一般，要求寻找所给数组的关键数字，首先先梳理一下题目说指出的逻辑

每个数所对应的中间数

3：5、8、4、2、1

5：8、4、2、1

6：3、5、8、4、2、1

7：11、17、26、13、20、10、5、8、4、2、1

8：4、2、1

11：17、26、13、20、10、5、8、4、2、1

通过观察上面的数据能发现：

5的中间数被3覆盖，所以可以舍弃5，继续观察发现3的中间数被6覆盖，所以可以舍弃3,在比较6和7时，发现6和3并没有被7所覆盖，因此6和7被保留，余下数字同理

于是逻辑如下：

当前数字在已存储的中间数中进行比对：

如果存在，说明当前数字的中间数已经被计算完成，于是跳过该数字

如果不存在，说明该数字没有被覆盖（没有被完全覆盖），继续计算该数字的中间数，每次计算后都与存储的中间数进行比对

如果存在，停止计算并比较当前的中间值是否与存储的关键字相同

  • 如果相同，删除之前的关键字，存储本数字
  • 如果不同，直接存储本数字即可

如果不存在，则存储该中间数，继续计算

于是代码如下：

#include<iostream>
#include<vector>
#include<set>
#include <algorithm>

using namespace std;

bool Callatz(int n,vector<int> &key,set<int> &num){
    if(find(key.begin(),key.end(),n) == key.end()){
        key.push_back(n);
        while(n != 1){
            if(n%2 == 0){
                n = n/2;
            }else{
                n = (n*3+1)/2;
            }
            if(find(key.begin(),key.end(),n) != key.end()){
                if(num.find(n) != num.end()){
                    num.erase(n);
                }
                return true;
            }else{
                key.push_back(n);
            }
        }
        return true;
    }else{
        return false;
    }
}

int main(){
    int k;
    int j=0;
    set<int> num;
    vector<int> key;
    cin >> k;
    for(int i=0;i < k;i++){
        int n;
        cin >> n;
        if(Callatz(n,key,num)){
            num.insert(n);
        }
    }
    for(auto it= num.rbegin();it != num.rend();it++){
        if(it == num.rbegin()){
            cout << *it;
        }else{
            cout << " " << *it;
        }
    }
    return 0;
}

总结一下

  • 在使用set，vector类型时，如果涉及其操作方法，需导入#include <algorithm>，否则会显示方法未定义
  • 面对可变数组时可以考虑一下vector，面对排序后输出时可以考虑一下set
  • set、vector在输出时不仅可以正向迭代输出，还可以反向迭代输出，只需将for循环中的begin、end换为rbegin、rend即可
