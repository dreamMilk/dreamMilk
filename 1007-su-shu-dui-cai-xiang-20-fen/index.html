1007 素数对猜想 (20分)

作者: Milk 's Home 日期: 2020-04-04 阅读时间:2.7 分钟 字数:666 分类: 代码练习 阅读量: -

题目

让我们定义dnd_ndn​为：dnd_ndn​=pn+1p_{n+1}pn+1​−pnp_npn​，其中pip_ipi​是第i个素数。显然有d1d_1d1​=1，且对于n> 1有dnd_ndn​是偶数。“素数对猜想”认为“存在无穷多对相邻且差为2的素数”。

现给定任意正整数N(<105)，请计算不超过N的满足猜想的素数对的个数。

输入格式：

输入在一行给出正整数N。

输出格式：

在一行中输出不超过N的满足猜想的素数对的个数。

输入样例1：

20

输出样例1：

4

思路：

题目思路比较开放，说一下根据这题所学到的数学知识

在检验是否为素数时，脑袋里的第一个想法就是暴力排查，寻找是否存在1和自己本身以外的约数

这种方法一般❌不建议使用，于是就发现了另外一个数学方法：埃拉托斯特尼筛法

要得到自然数n以内的全部素数，必须把不大于n\sqrt{n}n​的所有素数的倍数剔除，剩下的就是素数。

通过这个思想，我们可以将循环判断的范围很好地缩短，由O(n)变为O(log nlog\,nlogn)

在查找上述方法时发现还有一种思想可以继续精简循环范围：素数规律

素数只可能出现在6的倍数左右。

6的倍数就是6k，它附近的数就是6k-3,6k-2,6k-1,6k+1,6k+2,6k+3,那么不在6k左右的数是 6k-3,6k-2,6k+2,6k+3,第一个和最后一个数是可以整除3，另两个数是可以整除2的，所以他们肯定不是素数。

所以只有6的倍数左右的两个数才有可能是素数。

反例：比如25在6的倍数左右，但是它不是素数，

因此在6的倍数左右也不一定是素数，但是不在左右就一定不是素数

在最后还有一个是目前最快的素数算法：米勒-拉宾素性检验 (Miller–Rabin primality test)

该算法是基于费马小定理，时间复杂度为O(klog3nklog^3nklog3n)，因为其理论比较复杂，在此就不过多赘述了。

于是代码如下：

#include<iostream>

using namespace std;

bool judge(int i){
    if(i <= 3){
        return i > 1;
    }
    if(i%6 != 1 && i%6 != 5){
        return false;
    }
    for(int j=2;j*j <= i;j++){
        if(i%j == 0){
            return false;
        }
    }
    return true;
}

int main(){
    int num;
    int cont = 0;
    cin >> num;
    for(int i=5;i <= num;i++){
        if(judge(i) && judge(i-2)){
            cont ++;
        }
    }
    cout << cont << endl;
    return 0;
}
