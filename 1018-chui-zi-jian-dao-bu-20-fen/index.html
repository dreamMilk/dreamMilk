1018 锤子剪刀布 (20分)

作者: Milk 's Home 日期: 2020-04-16 阅读时间:3.9 分钟 字数:943 分类: 代码练习 阅读量: -

题目

大家应该都会玩“锤子剪刀布”的游戏：两人同时给出手势，胜负规则如图所示：

现给出两人的交锋记录，请统计双方的胜、平、负次数，并且给出双方分别出什么手势的胜算最大。

输入格式：

输入第 1 行给出正整数 N（≤105），即双方交锋的次数。随后 N 行，每行给出一次交锋的信息，即甲、乙双方同时给出的的手势。C 代表“锤子”、J 代表“剪刀”、B 代表“布”，第 1 个字母代表甲方，第 2 个代表乙方，中间有 1 个空格。

输出格式：

输出第 1、2 行分别给出甲、乙的胜、平、负次数，数字间以 1 个空格分隔。第 3 行给出两个字母，分别代表甲、乙获胜次数最多的手势，中间有 1 个空格。如果解不唯一，则输出按字母序最小的解。

输入样例：

10
C J
J B
C B
B B
B C
C C
C B
J B
B C
J J

输出样例：

5 3 2
2 3 5
B B

思路：

对输入的数字进行比较和统计，

对于统计胜负数据，只需收集一个人的数据即可，另外一个人的数据与其相反（一个人的胜利意味着另一个人的失败）

对于获胜手势的统计，使用长度为3的数组，因为统计的对象是固定数量的（剪刀、锤子、布）

本着能简单点就简单点的原则，在手势比较时进行了简化，因为布大于锤子对应的字母是B>C；锤子大于剪刀对应的字母是C>J；剪刀大于布对应的字母是J>B。字母小的一方获胜，只有剪刀和布的时候相反

用伪代码表示就是：if(手势1<手势2 || 手势1==J && 手势2==B) 手势1获胜

当我使用该逻辑提交运行时却发现有部分测试点无法通过，最后排查完毕后，发现是这个钻空子的逻辑出现了问题，因为输入的字符可能并没有规范的输入这三种字母，出现了其他情况，比如在游戏过程中，其中一方的手势出错了，那么这一局应该不计入成绩

不得不说PAT的测试点是真的太符合实际情况了，需要考虑各种需要处理的冲突，PAT之所以不给测试点数据，是因为不希望练习者只是为了通过测试点，使用各种钻空子的手法AC题目

这种想法在考试中可以尝试，但是在实际开发过程中不太可取，容易在后期留下难以修复的bug，以后还是多多思考一下题目逻辑，争取完美AC🏆

于是代码如下：

#include<iostream>

using namespace std;

int print(int m[3]){
    if(m[0] < m[1]){
        cout << (m[1]<m[2]?'J':'C');
    }else{
        cout << (m[0]<m[2]?'J':'B');
    }
    return 0;
}

int main() {
    int n;
    int mj[3]={0},my[3]={0};
    int lose=0,win=0;
    cin >> n;
    for(int i=0;i<n;i++){
        char s,t;
        cin >> s >> t;
        if (s == 'B' && t == 'C') {
            win++;
            mj[0]++;
        } else if (s == 'B' && t == 'J') {
            lose++;
            my[2]++;
        } else if (s == 'C' && t == 'B') {
            lose++;
            my[0]++;
        } else if (s == 'C' && t == 'J') {
            win++;
            mj[1]++;
        } else if (s == 'J' && t == 'B') {
            win++;
            mj[2]++;
        } else if (s == 'J' && t == 'C') {
            lose++;
            my[1]++;
        }
    }
    cout << win << " " << n-win-lose << " " << lose << endl;
    cout << lose << " " << n-win-lose << " " << win << endl;
    print(mj);
    cout << " ";
    print(my);
    return 0;
}
