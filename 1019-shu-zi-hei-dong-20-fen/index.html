1019 数字黑洞 (20分)

· 2020-04-17 · # 代码练习

题目

给定任一个各位数字不完全相同的 4 位正整数，如果我们先把 4 个数字按非递增排序，再按非递减排序，然后用第 1 个数字减第 2 个数字，将得到一个新的数字。一直重复这样做，我们很快会停在有“数字黑洞”之称的 6174，这个神奇的数字也叫 Kaprekar 常数。

例如，我们从6767开始，将得到

7766 - 6677 = 1089
9810 - 0189 = 9621
9621 - 1269 = 8352
8532 - 2358 = 6174
7641 - 1467 = 6174
... ...

现给定任意 4 位正整数，请编写程序演示到达黑洞的过程。

输入格式：

输入给出一个 (0,10410^4104) 区间内的正整数 N。

输出格式：

如果 N 的 4 位数字全相等，则在一行内输出 N - N = 0000；否则将计算的每一步在一行内输出，直到 6174 作为差出现，输出格式见样例。注意每个数字按 4 位数格式输出。

输入样例1：

6767

输出样例1：

7766 - 6677 = 1089
9810 - 0189 = 9621
9621 - 1269 = 8352
8532 - 2358 = 6174

输入样例2：

2222

输出样例2：

2222 - 2222 = 0000

思路：

题目描述的逻辑十分清楚，简单捋一下：

将输入的数进行降序和升序后计算差，然后再将差值重复上述步骤，直到出现6174或0为止

一开始尝试使用很原始的步骤：

数字使用数组存储并排序处理，然后使用递归的方式进行调用计算，结果不出意外地未通过，一方面是未考虑到题目的条件：输入给出一个 (0,10410^4104) 区间内的正整数 N（不一定是四位数）；另一方面是递归调用会导致栈溢出

最后通过查阅各路大神的代码，总结一下：

万能的sort函数：大部分的结构类型都能通过sort函数排序（默认升序）

能用现有函数实现的功能尽量不要使用原始方法实现

还有像往常一样不仔细看题，导致逻辑出现漏洞；当看完别人写的简洁代码时，感觉和别人写的都不是同一个题目，说明还要继续努力攀登✏️

于是代码如下：

#include<iostream>
#include<algorithm>

using namespace std;

bool cmp(char a,char b){
    return a>b;
}

int main() {
    string s;
    cin >> s;
    s.insert(0,4-s.length(),'0');
    int n;
    do {
        string n1=s,n2=s;
        sort(n1.begin(),n1.end(),cmp);
        sort(n2.begin(),n2.end());
        n = stoi(n1)-stoi(n2);
        s = to_string(n);
        s.insert(0,4-s.length(),'0');
        cout << n1 << " - " << n2 << " = " << s << endl;
    } while(n!=0 && n!=6174);
    return 0;
}
