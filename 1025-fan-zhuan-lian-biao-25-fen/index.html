1025 反转链表 (25分)

作者: Milk 's Home 日期: 2020-04-23 阅读时间:3.1 分钟 字数:743 分类: 代码练习 阅读量: -

题目

给定一个常数 K 以及一个单链表 L，请编写程序将 L 中每 K 个结点反转。例如：给定 L 为 1→2→3→4→5→6，K 为 3，则输出应该为 3→2→1→6→5→4；如果 K 为 4，则输出应该为 4→3→2→1→5→6，即最后不到 K 个元素不反转。

输入格式：

每个输入包含 1 个测试用例。每个测试用例第 1 行给出第 1 个结点的地址、结点总个数正整数 N (≤105)、以及正整数 K (≤N)，即要求反转的子链结点的个数。结点的地址是 5 位非负整数，NULL 地址用 −1 表示。

接下来有 N 行，每行格式为：

Address Data Next

其中 Address 是结点地址，Data 是该结点保存的整数数据，Next 是下一结点的地址。

输出格式：

对每个测试用例，顺序输出反转后的链表，其上每个结点占一行，格式与输入相同。

输入样例：

00100 6 4
00000 4 99999
00100 1 12309
68237 6 -1
33218 3 00000
99999 5 68237
12309 2 33218

输出样例：

00000 4 33218
33218 3 12309
12309 2 00100
00100 1 99999
99999 5 68237
68237 6 -1

思路：

题目模拟链表存储过程，所有的地址均为虚拟地址，并非实际地址：

对于链表结点的结构，可以分为结构体或多个数组进行存储表示

因为题目涉及到多次交换位置，使用多个数组进行操作可能会比较方便进行定位翻转

于是有数据数组、下一个结点地址数组以及整理好顺序的结点地址数组

数据输入时，将数据值存放到对应地址下表的数组中，下一节点的地址存放到对应地址下标数组中

之所以需要整理链表，一是捋顺链表顺序，二是排除干扰结点（除头节点外没有前结点的结点）

整理完成的数组链表只包含有效结点的地址（根据地址可以查询数值和下一节点地址）

然后将新的数组链表进行定长翻转，进行输出即可

一开始没有理解题目意思，认为地址是真实地内存地址，十分不理解，最后还是参考他人的逻辑进行编写

于是代码如下：

#include<iostream>
#include<algorithm>

using namespace std;

int main() {
    int start,n,k,address;
    cin >> start >> n >> k;
    int data[100000],next[100000],new_list[100000];
    for(int i=0;i<n;i++){
        cin >> address;
        cin >> data[address] >> next[address];
    }
    int cont=0;
    while(start != -1){
        new_list[cont++]=start;
        start = next[start];
    }
    for(int i=0;i<cont-(cont%k);i += k){
        reverse(begin(new_list)+i,begin(new_list)+i+k);
    }
    for(int i=0;i<cont-1;i++){
        printf("%05d ",new_list[i]);
        cout << data[new_list[i]];
        printf(" %05d\n",new_list[i+1]);
    }
    printf("%05d ",new_list[cont-1]);
    cout << data[new_list[cont-1]];
    printf(" -1\n");
    return 0;
}
