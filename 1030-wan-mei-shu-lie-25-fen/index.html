1030 完美数列 (25分)

作者: Milk 's Home 日期: 2020-04-28 阅读时间:2.3 分钟 字数:560 分类: 代码练习 阅读量: -

题目

给定一个正整数数列，和正整数 p，设这个数列中的最大值是 M，最小值是 m，如果 M≤mp，则称这个数列是完美数列。

现在给定参数 p 和一些正整数，请你从中选择尽可能多的数构成一个完美数列。

输入格式：

输入第一行给出两个正整数 N 和 p，其中 N（≤10510^5105）是输入的正整数的个数，p（≤10910^9109）是给定的参数。第二行给出 N 个正整数，每个数不超过 10910^9109。

输出格式：

在一行中输出最多可以选择多少个数可以用它们组成一个完美数列。

输入样例：

10 8
2 3 20 4 5 1 6 7 8 9

输出样例：

8

思路：

题目寻找最大完美数列，第一个想法就是滑动窗口：

先将数列从小到大排序，设定初始窗口大小为整个数列

如果末尾两个数的差大于头部两个数的差与参数的积，窗口尾部前移一格；反之，头部后移一格

v[last]-v[last-1] > p*(v[first+1]-v[first])

直到满足M≤mp或头尾相等时，结束循环，窗口的长度即为结果

但是运行后其他测试点完美通过，而测试点4显示的是答案错误：

百度了不少人的代码，发现大多数都是两次for循环进行求解，并且都是测试点4出现超时问题，然而我运行时显示答案错误？？还是不清楚这个逻辑的错误出现在什么地方

等以后再回过头来仔细梳理一下，感觉这个思路也不比双重for循环差，为啥没什么人写这个解题思路😫

于是代码如下：

#include<iostream>
#include<algorithm>
#include<vector>

using namespace std;

int main() {
    int n;
    long long int p;
    cin >> n >> p;
    vector<int> v(n);
    for(int i=0;i<n;i++){
        cin >> v[i];
    }
    sort(v.begin(),v.end());
    int last = v.size()-1;
    int first = 0;
    while(v[last] >v[first]*p && last!=first){
        if(v[last]-v[last-1] > p*(v[first+1]-v[first])){
            last --;
        }else first++;
    }
    cout << last-first+1;
    return 0;
}
