1039 到底买不买 (20分)

2020-05-06 Views 代码练习725字3 min read繁

题目

小红想买些珠子做一串自己喜欢的珠串。卖珠子的摊主有很多串五颜六色的珠串，但是不肯把任何一串拆散了卖。于是小红要你帮忙判断一下，某串珠子里是否包含了全部自己想要的珠子？如果是，那么告诉她有多少多余的珠子；如果不是，那么告诉她缺了多少珠子。

为方便起见，我们用[0-9]、[a-z]、[A-Z]范围内的字符来表示颜色。例如在图1中，第3串是小红想做的珠串；那么第1串可以买，因为包含了全部她想要的珠子，还多了8颗不需要的珠子；第2串不能买，因为没有黑色珠子，并且少了一颗红色的珠子。

输入格式：

每个输入包含 1 个测试用例。每个测试用例分别在 2 行中先后给出摊主的珠串和小红想做的珠串，两串都不超过 1000 个珠子。

输出格式：

如果可以买，则在一行中输出 Yes 以及有多少多余的珠子；如果不可以买，则在一行中输出 No 以及缺了多少珠子。其间以 1 个空格分隔。

输入样例1：

ppRYYGrrYBR2258
YrR8RrY

输出样例1：

Yes 8

输入样例2：

ppRYYGrrYB225
YrR8RrY

输出样例2：

No 2

思路：

题目又是进行字符串比较：

一个统计原字符串字符出现频率的数组：str1[256]，这是为了方便直接使用256，容纳所有ASCII码字符

循环对比的字符串，如果str1中存在，所在下标的数值减一，否则缺少值cont加一

如果最后没有缺少，直接输出原字符串与对比字符串字符相差个数即可

一开始死脑筋，只想按照要求存储[0-9]、[a-z]、[A-Z]，结果代码写的比较啰嗦：

发现开辟256空间虽然有一点浪费，但是写出来的代码却清爽了不少

关于变量问题，如果写的是全局变量，默认值一般为0；如果写的是局部变量，默认值一般为随机值：

全局变量、静态变量处于数据区；而局部变量处于堆栈区，为当时内存中的值

不要利用编译器的这种默认原则
而是应当显示的指定
必要的初始化不是笨拙
而是规范

于是代码如下：

#include<iostream>

using namespace std;
int main() {
    string s1,s2;
    cin >> s1 >> s2;
    int str1[256]={0};
    for(int i=0;i<s1.length();i++){
        str1[s1[i]] ++;
    }
    int cont=0;
    for(int i=0;i<s2.length();i++){
        if(str1[s2[i]] == 0) cont ++;
        else str1[s2[i]] --;
    }
    if(cont ==0) cout << "Yes" << " " << s1.length()-s2.length();
    else cout << "No" << ' ' << cont;
    return 0;
}
