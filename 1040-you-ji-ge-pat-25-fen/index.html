1040 有几个PAT (25分)

题目

字符串 APPAPT 中包含了两个单词 PAT，其中第一个 PAT 是第 2 位(P)，第 4 位(A)，第 6 位(T)；第二个 PAT 是第 3 位(P)，第 4 位(A)，第 6 位(T)。

现给定字符串，问一共可以形成多少个 PAT？

输入格式：

输入只有一行，包含一个字符串，长度不超过10510^5105，只包含 P、A、T 三种字母。

输出格式：

在一行中输出给定字符串中包含多少个 PAT。由于结果可能比较大，只输出对 1000000007 取余数的结果。

输入样例：

APPAPT

输出样例：

2

思路：

题目核心问题是包含多少个PAT，可以使用排列组合的思想：

为了保障所得字符顺序是正确的，只能使用字符A，然后统计其他字符的所有排列组合

（如果选择的是P、T的话，排列组合的个数不能通过简单的字符数相乘得到，因为无法保证该组合是PAT）

在循环字符串时遇到A时，将前面P的个数与后面T的个数相乘，得到当前A字符的所有排列可能，想要得到结果，只需要将所有字符A的排列可能相加即可

在统计字符数时：

在最开始将字符P的个数P置为0，字符T的个数T为字符串中所有T的个数

每遇到字符P就P++，遇到字符T就T--，遇到字符A就进行P*T，最后输出积的累计值

在计算时，只要是计算后可能超出int范围的值都需要进行取模操作，防止数据溢出，至于为什么需要取余，而且为什么是对1000000007取余，这个就需要自行谷歌百度了

于是代码如下：

#include<iostream>
#include<algorithm>

using namespace std;

int main() {
    string s;
    cin >> s;
    int sum=0,p=0,t=count(s.begin(),s.end(),'T');
    for(int i=0;i < s.length();i++){
        if(s[i]=='P') p++;
        else if(s[i]=='A') sum = (sum+(p*t)%1000000007)%1000000007;
        else t--;
    }
    cout << sum;
    return 0;
}
