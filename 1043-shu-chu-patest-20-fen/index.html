1043 输出PATest (20分)

题目

给定一个长度不超过 104 的、仅由英文字母构成的字符串。请将字符重新调整顺序，按 PATestPATest.... 这样的顺序输出，并忽略其它字符。当然，六种字符的个数不一定是一样多的，若某种字符已经输出完，则余下的字符仍按 PATest 的顺序打印，直到所有字符都被输出。

输入格式：

输入在一行中给出一个长度不超过 104 的、仅由英文字母构成的非空字符串。

输出格式：

在一行中按题目要求输出排序后的字符串。题目保证输出非空。

输入样例：

redlesPayBestPATTopTeePHPereatitAPPT

输出样例：

PATestPATestPTetPTePePee

思路：

题目依然是字符统计问题：

使用一个数组存储PATest每个字符出现的次数

然后一直循环该数组，直到数组中所有值为0

看了一些其他人的代码，思路基本相同：

使用map结构存储字符串所有字符出现次数，然后循环map中PATest的各个字符，直到这些数值全为0

代码看着确实十分简洁，也有挺多人模仿那种写法，但是（当前情况下）使用map的内存开销肯定会大于两个数组，并且只是使用map的键值对特性，没有使用其他特性，而这个使用数组是可以实现的

或许当事人只是为了追求代码简洁，使用了这种写法，我也没有批判这种写法不对，只是希望提醒自己不要因为大众写法而影响自身

挣扎了很久还是坚持这种冗长的写法，好的代码肯定会学习，但不会盲目的模仿他人的代码，多分析自己代码与他人的差距，学会独立思考🤓

于是代码如下：

#include<iostream>

using namespace std;

int main() {
    string s;
    cin >> s;
    int arr[6]={0};
    char c[6]={'P','A','T','e','s','t'};
    for(int i=0;i<s.length();i++){
        switch (s[i]){
            case 'P': arr[0]++;break;
            case 'A': arr[1]++;break;
            case 'T': arr[2]++;break;
            case 'e': arr[3]++;break;
            case 's': arr[4]++;break;
            case 't': arr[5]++;break;
        }
    }
    bool flag=true;
    while(flag){
        flag = false;
        for(int i=0;i<6;i++){
            if(arr[i]-- > 0) {cout << c[i];flag = true;}
        }
    }
    return 0;
}
