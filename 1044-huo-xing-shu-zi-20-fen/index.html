1044 火星数字 (20分)

· 2020-05-11 · # 代码练习

题目

火星人是以 13 进制计数的：

  • 地球人的 0 被火星人称为 tret。
  • 地球人数字 1 到 12 的火星文分别为：jan, feb, mar, apr, may, jun, jly, aug, sep, oct, nov, dec。
  • 火星人将进位以后的 12 个高位数字分别称为：tam, hel, maa, huh, tou, kes, hei, elo, syy, lok, mer, jou。

例如地球人的数字 29 翻译成火星文就是 hel mar；而火星文 elo nov 对应地球数字 115。为了方便交流，请你编写程序实现地球和火星数字之间的互译。

输入格式：

输入第一行给出一个正整数 N（<100），随后 N 行，每行给出一个 [0, 169) 区间内的数字 —— 或者是地球文，或者是火星文。

输出格式：

对应输入的每一行，在一行中输出翻译后的另一种语言的数字。

输入样例：

4
29
5
elo nov
tam

输出样例：

hel mar
may
115
13

思路：

题目需要字符转数字，数字转字符：

在结构上，一开始不知道是选用map结构还是数组结构好，但是感觉map在键值对存储上应该好一点

在字符转数字方面确实比较顺利，但是在数字转字符时，因为map存储时乱序的原因，发现两个字符表达的数，需要通过遍历两边map进行确定，但是分为两个map结构就没意义了

最后为了节省还是选择使用两个数组进行存储：

数组主要是在字符转数字时，需要遍历两次数组，（这不是疯了吗，不就是为了节省时间才选择的数组吗）

但是两个数组可以同时开始遍历，比起两次map结构遍历还是快一点

写完发现数组遍历写了两遍，感觉与一次遍历两遍map感觉没什么区别🤦‍♂️

其实整体写下来发现两种写法各有优缺点：map结构在字符转数字时，比起数组会好一点；数组结构在数字转字符时，比起map结构会好一点。

于是代码如下：

#include<iostream>
#include<string>

using namespace std;

int main() {
    int n;
    cin >> n;
    string v1[13]={"tret","jan","feb","mar","apr","may","jun","jly","aug","sep","oct","nov","dec"};
    string v2[13]={" ","tam","hel","maa","huh","tou","kes","hei","elo","syy","lok","mer","jou"};
    getchar();
    for(int i=0; i<n; i++) {
        string input;
        getline(cin,input);
        //两个字符串的火星文
        if(input.length()>3){
            int num=0;
            //分别检索两个字符串的含义
            for(int i=0;i<13;i++){
                if(v2[i]==input.substr(0,3)) num += i*13;
                if(v1[i]==input.substr(4)) num += i;
            }
            cout << num << endl;
        }
        //一个字符串的火星文
        else if(islower(input[0])){
            //分别从高位字符数组和低位字符数组中检索
            for(int i=0;i<13;i++){
                if(v1[i]==input){cout << i << endl;break;}
                if(v2[i]==input){cout << i*13 << endl;break;}
            }
        }
        //数字
        else{
            int num=stoi(input);
            int n1=num/13,n2=num%13;
            if(n1*n2!=0)cout << v2[n1]  << ' ' << v1[n2];
            else n1 > 0 ? cout << v2[n1]:cout << v1[n2];
            cout << endl;
        }
    }
    return 0;
}
